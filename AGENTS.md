# xiuxiuyingyu-site Agent Guide

本文件作用域覆盖整个 `xiuxiuyingyu-site` 项目。

后续接手本项目的 AI，进入仓库后先读本文件，再按任务类型继续读取对应说明。
目标是减少重复犯错，直接沿用已经确认过的产品要求和处理方式。

## 项目定位

- 这是一个给小朋友用的英语学习项目。
- 交互要简单、直接、稳定，优先保证“能学、能听、能复习”。
- 文案和释义尽量儿童化、短句化，不要写成成人词典风格。

## 先读哪些文件

### 1. 任何涉及题目、词表、播放策略、错题逻辑的修改

先读：

- `docs/product-conventions.md`
- `docs/maintenance-lessons.md`

### 2. 任何涉及音频生成、慢速音频、本地 TTS、参考音频的修改

先读：

- `docs/product-conventions.md`
- `docs/audio-generation-manual.md`
- `docs/maintenance-lessons.md`

## 当前关键约定

- 正常播放：走统一的美式词典接口
- 慢速播放：走本地 `IndexTTS2` 生成的自然慢速音频
- 不允许前端用 `playbackRate` 伪造慢速
- 更新每日词汇时，必须同时更新听写和日常学习两个入口

## 用户新要求优先级

- 如果用户最新要求和说明文件冲突，以用户最新要求为准。
- 如果用户没有改规则，就沿用说明文件里的既定做法，不要反复追问。

## 每次修改后的最低校验

默认至少执行：

- `npm run lint`
- `npm run test`
- `npm run build`

如果改了音频或播放逻辑，再补一轮人工确认：

- 至少试听 1 个正常播放单词
- 至少试听 1 个慢速播放单词
- 确认页面里的词表、题目、音频文件名一致

## 修改原则

- 优先做最小改动，不要顺手重构无关页面。
- 本项目最容易出错的地方不是 React 代码，而是“词表、题目、音频、播放策略”四者不同步。
- 先保证同步，再考虑优化。

## 经验记录要求

- 只要本次任务改了代码、音频、页面交互、题目或配置，就要更新 `docs/maintenance-lessons.md`。
- 记录内容至少包含：
  - 问题现象
  - 根因判断
  - 最终处理
  - 验证方式
  - 给后续修改者的提醒
- 不要只记录“做了什么”，要写清“为什么会错、这次怎么避免下次再错”。
