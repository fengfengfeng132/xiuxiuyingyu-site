# 音频生成手册

本文件记录这个项目在这台机器上生成和替换单词音频的稳定做法。  
后续 AI 只要涉及 `IndexTTS2`、慢速音频、本地音频替换，就先读这里。

## 本机环境

当前机器上已使用过、并验证可生成的本地环境：

- `D:\AI\index-tts-nolfs`

## 生成音频时的总原则

- 先在仓库里的 `tmp/` 下生成临时文件
- 全部生成成功后，再整体替换到最终目录
- 最后删除 `tmp/`

这样即使中途失败，也不会把现有可用音频破坏掉。

## 目标目录

默认同时更新两套资源：

- 普通版：`public/audio/words/us/*.wav`
- 慢速版：`public/audio/words/us-slow/*.wav`

即使当前正常播放主要走词典接口，本地普通版也建议同步更新，作为统一声音的备用资源。

## 参考音频选择

### 首选提示源：固定的系统女声参考音频

当前这台机器上，优先使用：

- `Microsoft Zira Desktop`（`en-US`，女声）

推荐先用仓库里的脚本生成两份固定提示音频：

- 正常版参考提示
- 慢速版参考提示

脚本：

- `tools/New-IndexTtsReferencePrompts.ps1`

这样做比“拿旧的单词音频继续拼接当提示源”更稳定，因为不会把上一轮生成里的伪影继续放大。

### 不要直接依赖 `examples/voice_01.wav`

在这台机器上，以下文件不是稳定可用的参考音频来源：

- `D:\AI\index-tts-nolfs\examples\voice_01.wav`

读取时可能报：

- `Format not recognised`
- `audioread.exceptions.NoBackendError`

### 备选做法

如果当前机器上没有可用的 `Microsoft Zira Desktop`，再退回到下面这个办法：

直接使用项目里已有的本地词音频，拼接成参考提示音频：

- 普通版参考音频：从 `public/audio/words/us/` 中挑若干稳定 `.wav`
- 慢速版参考音频：从 `public/audio/words/us-slow/` 中挑若干稳定 `.wav`

这条路能工作，但长期反复使用会有“递归自克隆”的风险，容易把回音、拖尾、节奏漂移一轮轮带下去。

## 慢速音频的硬规则

- 不要用前端 `playbackRate < 1`
- 不要做会引入伪影的时间拉伸
- 让 `IndexTTS2` 直接生成自然慢速版

也就是说，慢速必须是模型原生生成，而不是浏览器拉慢。

## 遇到“回音 / 拖尾 / 不够清爽”时的排查顺序

优先检查：

1. 有没有做网页端慢放
2. 参考音频是否干净
3. 是否误用了不一致的音频源
4. 是否加了额外后处理

不要第一时间加新的音频后处理。  
先确认不是“参考音频脏”或“前端拉速”的问题。

## 这台机器上的常见工具问题

### 1. `ffmpeg.exe` 可能不可直接用

当前环境下，通过 WinGet 链接的 `ffmpeg.exe` 可能会报：

- “没有应用程序与此操作的指定文件有关联”

如果只是做简单拼接或检查：

- 优先用 Python 标准库 `wave`

### 2. `rg.exe` 可能不可直接用

当前环境下，通过 WinGet 链接的 `rg.exe` 也可能不可直接启动。

替代方式：

- `Get-ChildItem`
- `Select-String`

不要因为工具不可用就停住。

## 生成完成后的检查项

至少确认：

1. `public/audio/words/us/` 文件名与当前词表一致
2. `public/audio/words/us-slow/` 文件名与当前词表一致
3. 旧词音频已被替换，不要留下和当前词表无关的旧文件
4. 页面里的词表、题目、音频文件名一致
5. 慢速版时长不要明显短于普通版
6. 尾部能量不要异常高，否则容易出现“抖尾音”或拖尾感

## 建议的稳定化流程

生成单词音频时，优先采用下面这套顺序：

1. 先用 `tools/New-IndexTtsReferencePrompts.ps1` 生成固定女声提示音频
2. 用固定提示音频跑 `IndexTTS2`
3. 对重点单词至少生成 2 到 3 个候选
4. 优先保留：
   - 本地 ASR 能识别出目标单词的候选
   - 尾部更干净的候选
   - 慢速版时长明显大于普通版的候选

不要“一次生成直接入库”。

## 最后校验

改完音频或词表后，默认执行：

- `npm run lint`
- `npm run test`
- `npm run build`

并至少试听：

- 1 个正常播放单词
- 1 个慢速播放单词
